Task4 Notification Handler

For this task, I add an additional function in jupyter lab called time_handler. It first converts the binary date into a string and then split it by finding “T:”. The example shown here indicates how it is performed in in the previous GET_TIME_MILLIS, and as we can see, it can print the time without the print command.


        def time_handler(uuid,byte_array):
            string_value = byte_array.decode("utf-8")
            print("Received string:", string_value)
            if "T:" in string_value:
                time_millis = string_value.split(":")[1]
                print(f"Extracted Time: {time_millis} ms")
        ble.start_notify(ble.uuid['RX_STRING'], time_handler)
        ble.send_command(CMD.GET_TIME_MILLIS, "")

Task8 Comparison

The first method sends data in real-time but is slower; the second collects data first then sends it super fast (some timestamps repeat because it crams points into 1ms). First method pros: real-time updates/debugging; cons: slow. Second method pros: fast transfer; cons: not real-time (bad for fast-moving robots) + uses Artemis storage. Use first for live robot updates/debugging; second for recording stunts/sensor data for later analysis. Artemis has 384,000 bytes of RAM. With time/temp as 4-byte floats, it can store ~96,000 paired values max.
